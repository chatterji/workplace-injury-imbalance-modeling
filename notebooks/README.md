# Notebook Guidance

This repository is intentionally centered on reusable Python modules instead of a single experimental notebook.

If you want to add a notebook later for recruiters or interviewers, create one polished walkthrough notebook such as:

`01_workplace_injury_claim_modeling_walkthrough.ipynb`

Recommended notebook content:
- business problem
- class imbalance summary
- modeling methods compared
- final results table
- business interpretation of precision/recall tradeoffs

Avoid including:
- exploratory dead ends
- tutorial-style borrowed text
- confidential data extracts
